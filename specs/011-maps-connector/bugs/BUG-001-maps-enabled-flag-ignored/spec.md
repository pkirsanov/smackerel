# Spec: BUG-001 — MAPS_ENABLED Flag Ignored

**Parent:** [011-maps-connector](../../spec.md)
**Type:** Bug fix
**Priority:** P1

## Problem

The `MAPS_ENABLED` env var generated by the SST pipeline is dead code — `config.Config` never reads it, and `connectors.go` never checks it. The maps connector starts whenever `MAPS_IMPORT_DIR` is non-empty, regardless of the `enabled` flag.

## Requirements

- R-BUG001-01: `config.Config` MUST include a `MapsEnabled bool` field loaded from `MAPS_ENABLED`
- R-BUG001-02: Maps connector auto-start MUST check `cfg.MapsEnabled` before starting
- R-BUG001-03: Guard pattern MUST match bookmarks/browser-history (`Enabled && Dir != ""`)
- R-BUG001-04: Existing unit tests MUST continue to pass
