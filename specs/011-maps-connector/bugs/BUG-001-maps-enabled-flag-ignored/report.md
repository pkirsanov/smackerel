# Report: BUG-011-001 — MAPS_ENABLED Flag Ignored

Links: [spec.md](spec.md) | [scopes.md](scopes.md)

---

## Summary

Devops sweep finding DEVOPS-011-F1 (MEDIUM): the `MAPS_ENABLED` env var generated by the SST pipeline was dead code. `config.Config` never read it, and `cmd/core/connectors.go` started the maps connector whenever `MAPS_IMPORT_DIR` was non-empty regardless of the user's `enabled: false` setting in `config/smackerel.yaml`. This violated the bookmarks / browser-history connector pattern.

Fix added a `MapsEnabled bool` field to `config.Config`, loaded from `MAPS_ENABLED`, and changed the auto-start guard in `cmd/core/connectors.go` from `cfg.MapsImportDir != ""` to `cfg.MapsEnabled && cfg.MapsImportDir != ""`. Connector behavior unchanged when enabled.

## Completion Statement

Scope 01 DoD items verified. Field present at `internal/config/config.go:76`, env loader at `:269`, guard at `cmd/core/connectors.go:120`. Unit tests `TestLoad_ConnectorPathFields` and `TestLoad_ConnectorPathFieldsOptional` cover both true and false MAPS_ENABLED states and PASS. `go build ./cmd/core` compiles cleanly.
