### Summary

BUG-001 was filed 2026-04-26 by `bubbles.workflow` (bugfix-fastlane). The bot's
duplicate-URL reply at [internal/telegram/share.go](../../../../internal/telegram/share.go)
claimed it had merged the new context onto the existing artifact, but the duplicate
path in [internal/api/capture.go](../../../../internal/api/capture.go) returned
`409 DUPLICATE_DETECTED` without persisting `req.Context`, making the reply
structurally untruthful and breaking [SC-TSC04](../../spec.md) plus the parent
uservalidation acceptance item "Duplicate URL share merges new context without
re-processing".

The fix introduced `pipeline.MergeUserContext` in
[internal/pipeline/merge.go](../../../../internal/pipeline/merge.go) and wired it into
`Processor.Process` in [internal/pipeline/processor.go](../../../../internal/pipeline/processor.go)
so a `*DuplicateError` with non-empty `req.Context` performs a JSONB
`metadata.user_contexts` append on the existing artifact row before re-raising the
duplicate error. The API response shape and the bot reply text are unchanged; only
the side-effect on the existing row is added.

**Claim Source:** `interpreted` from the spec, design, and code anchors above; no
runtime claims are made in this section.
