# User Validation: 074 Capture-as-Fallback Cross-Cutting Policy

## Checklist

- [x] Planning baseline reflects `spec.md` and `design.md` scenarios SCN-074-A01 through SCN-074-A11.
- [ ] User can tell an unhandled thought was saved without being asked to tag or organize it.
- [ ] Repeating the same thought quickly returns an already-captured acknowledgement without creating a duplicate Idea.
- [ ] Explicit capture and fallback capture are visibly/auditably distinct by provenance.
- [ ] Operator can trace fallback capture telemetry from cause counter to IntentTrace to Idea artifact id.
- [ ] Telegram, HTTP, WhatsApp, web, and Android render the same saved-as-idea shape and copy.

## Planning Note

This checklist is a user-acceptance scaffold. Items other than the planning baseline require implementation and validation evidence before a human reviewer marks them complete.
