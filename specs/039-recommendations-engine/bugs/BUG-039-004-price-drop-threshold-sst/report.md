# Report: BUG-039-004 ÔÇö price-drop threshold fallback Ôćĺ fail-loud SST

**Workflow mode:** `bugfix-fastlane` (parent-expanded ÔÇö the active runtime lacks `runSubagent`)
**Owner:** `bubbles.workflow`
**Resolved:** 2026-06-07

## Summary

The price-drop watch evaluator used a hardcoded `thresholdPct = 0.10` as the
final fallback when neither the trigger nor the user's watch filter supplied
`threshold_pct` ÔÇö a binding NO-DEFAULTS / smackerel-no-defaults violation and the
one remaining hardcoded operational constant from the owner's "NO const limits"
sweep. The fallback now comes from fail-loud SST
(`recommendations.watches.default_price_drop_threshold_pct`); the trigger and
user-filter sources still take precedence.

## Root Cause

A Go literal `0.10` was the last level of the threshold-resolution precedence.

## Fix

New fail-loud SST key + a `resolvePriceDropThreshold(trigger, watch, defaultPct)`
helper (trigger Ôćĺ filter Ôćĺ SST default) threaded through the Evaluator; the
literal is removed.
